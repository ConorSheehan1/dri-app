Author: Damien Gallagher <damien.gallagher@nuim.ie>

    Fixed a bug where mp2 files were not being characterized by FITS. We have
    to use a workaround where the mp2 file is copied into a temp file with an mp3 extension.
    
    Added a lot of new Solr fields based on the characterization data provide by FITS.
    
    Added two new fields Solr fields to replace our dependency on using metadata type fields
    for generating file icons etc:
    
    - file_type_tesim
      This an array. It can be either ["collection"], ["unknown"], or an array of following four
      file types : "audio", "video", "image" or "text".
    
    - file_type_display_tesim
      This is an array that should be identical in length to file_type_tesim. It contains the
      text that should be displayed to the end user. For example, an element called "collection" in
      file_type_tesim may correspond to a "Series" element in file_type_display_tesim. The elements
      in both file_type_tesim and file_type_display_tesim are ordered so that the correspond to each
      other.
    
    These fields are derived from data stored in GenericFile. So once a GenericFile is characterized, a
    queued task is now sent to update the index of the Batch file it belongs to.
    
    You will need to reindex the entire repository to see the changes. Avoid using index_everything as
    the indexing has to be done in a specific order, so that the GenericFile objects are indexed before
    the Batch objects. Please, use the following commands in order:
    
    GenericFile.all.each { |obj| obj.update_index }
    Batch.all.each { |obj| obj.update_index }
